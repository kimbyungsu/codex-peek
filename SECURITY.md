# 보안 정책 (Security)

codex-peek / Codex Bridge가 **무엇을 보호하고, 무엇을 보호하지 않는지**를 분명히 적습니다. 과장하지 않는 것이 목표입니다.

> 아래 경로의 `~/.codex`(코덱스 홈)·`~/.codex-bridge`(브릿지 홈)·`~/.claude`(Claude 설정 폴더)는 모두 **기본값**이며, 각각 `$CODEX_HOME`·`$CODEX_BRIDGE_HOME`·`$CLAUDE_CONFIG_DIR`로 바뀔 수 있습니다. 자세한 경로·데이터 목록은 [PRIVACY.md](PRIVACY.md) 참고.

## 보안 경계인 것

- **로컬 전용 동작**: 이 도구는 당신 컴퓨터 안에서만 동작합니다. 코덱스 호출은 당신이 이미 설치한 `codex` 명령을 로컬에서 실행하는 것이고, 별도의 서버로 데이터를 보내지 않습니다. (네트워크 호출 코드가 없습니다.)
- **코덱스 원본 대화는 기본 읽기 전용**: 코덱스 세션 기록(`~/.codex/sessions/**/rollout-*.jsonl`)은 평소 **읽기만** 합니다. 세션을 "숨김" 처리해도 원본을 지우지 않고 별도 파일(`~/.codex-bridge/sessions-meta.json`)에 표시만 합니다. **단 하나의 예외**: 대시보드에서 당신이 직접 "영구 삭제" 버튼을 눌러 확인하면 그 rollout 파일을 삭제합니다(되돌릴 수 없음). 이것은 당신이 명시적으로 누른 경우에만 일어나며, 그 세션을 가리키던 모든 연결도 함께 정리됩니다.
- **설정 파일 손상 방지**: 설치기가 Claude 설정(`~/.claude/settings.json`)에 검증 훅을 끼워 넣을 때, 먼저 `.bak` 백업을 만들고, 형식이 예상과 다르면 **건드리지 않고 중단**합니다. 다른 사람이 넣은 훅은 보존합니다.

## 보안 경계가 *아닌* 것 (중요)

- **`codex` 직접 호출 차단은 "샌드박스"가 아니다**: PreToolUse 훅(`codex-guard.js`)은 에이전트가 정책층(계약·검증)을 건너뛰고 날것의 `codex`를 부르지 않도록 **유도**하는 장치입니다. 악의적 사용자나 프로세스가 우회하는 것을 막는 권한 격리가 아닙니다. 진짜 격리가 필요하면 OS 수준의 프로세스/권한 통제가 별도로 필요합니다.
- **훅은 당신 권한으로 도는 로컬 스크립트**: 설치기가 등록하는 4개 훅(매 입력 시 계약 주입, Bash 가드, 플랜 확정 게이트[3트랙 프로젝트 기본 켜짐(2026-07-09 승격) — 지도 없음/낡음 시 세션당 2회까지 차단 후 통과·오류 시 fail-open·scope-gate CLI로 끄기 가능, 2트랙은 관측 기록만], 완료 시 검증 가드)은 당신 사용자 권한으로 Node 스크립트를 실행합니다. 신뢰하는 환경에서만 설치하세요.
- **계약/지침 텍스트는 강제 실행이 아니라 모델에게 주는 지시**: 주입되는 규칙은 모델이 따르도록 *지시*하는 텍스트이지, 위반을 물리적으로 막는 장치가 아닙니다.

## 설치기가 바꾸는 것

- `~/.claude/settings.json` — 검증·게이트 훅 4개 병합(`.bak` 백업 후, 기존 훅 보존)
- `~/.codex-bridge/` — 브릿지 스크립트 복사
- VS Code 확장 설치(`code --install-extension`)

되돌리려면: 설치기의 훅 제거 경로를 쓰거나 `settings.json`에서 해당 훅을 지우고, `~/.codex-bridge/` 폴더를 삭제하고, 확장을 제거하면 됩니다.

## 취약점 신고

보안 문제는 공개 이슈로 올리기 전에 저장소 소유자에게 비공개로 알려 주세요(GitHub: kimbyungsu/codex-peek). 재현 절차·영향 범위·환경(OS·Node·VS Code·Codex 버전)을 함께 적어 주시면 빠릅니다.
