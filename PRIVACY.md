# 개인정보·데이터 정책 (Privacy)

핵심: **이 확장 자체가 자체 서버로 보내는 데이터는 없습니다.** 외부로 나가는 경로는 정확히 세 갈래입니다. ⑴ **DeepSeek 키를 등록한 경우에만**: ① 지도 생성 실행 시 자료 꾸러미 전송(당신이 직접 실행하거나, 3트랙 자동 지시를 받은 Claude가 실행 — 아래 "자동 지시" 참조) ② 3트랙을 켤 때 연결 점검 요청 1회(꾸러미 아님 — 접속 확인용 최소 요청). ⑵ **기본 정찰(키 불필요)**: 지도 생성이 실행되면 같은 자료 꾸러미가 **당신이 이미 쓰는 Claude CLI를 통해 Claude 서비스로** 전달됩니다(별도 결제 없음 — 쓰시던 구독 범위. 검증 요청이 Codex로 가는 것과 같은 성격의, 이미 동의된 AI 통신 경로). ⑶ **Codex 정찰(대시보드에서 탐색 담당을 Codex로 선택한 경우)**: 지도 생성이 실행되면 같은 자료 꾸러미가 **당신이 이미 쓰는 codex CLI를 통해 Codex 서비스로** 전달됩니다(검증과 분리된 독립 실행 1회·읽기 전용 강제·세션 기록 무잔재 — 쓰시는 Codex 계정 사용량 범위). 전송 내용·제외 목록은 아래 "외부로 나가는 것". **DeepSeek 키가 없으면 이 확장이 새로 여는 외부 통신은 없습니다**(기본 정찰·Codex 정찰·검증은 당신이 이미 쓰는 Claude/Codex CLI의 통신). **자동 의미 보강(P8)**: 3트랙에서 '의미 보강 담당'을 정하고 **자동 실행에 별도 동의**하면(기본 Claude 담당=대시보드 1클릭 동의·DeepSeek/Codex/자동형=담당 선택 시 모달 동의 — 이전에 저장해 둔 선택이나 3트랙 동의만으로는 절대 자동 실행되지 않음), 새 지도·소스 변경이 관측될 때 지도 초안과 소스 발췌(파일당 최대 4,000자·최대 20파일)가 위 세 갈래 중 해당 담당 경로로 전송됩니다. 동의를 지우면(담당 변경·철회) 다음 실행부터 멈춥니다. 아래는 정확히 무엇을 읽고, 무엇을 쓰는지입니다.

> **경로 표기**: 아래 경로들은 환경변수로 바뀔 수 있고, 없을 때만 기본값을 씁니다. 그래서 고정이 아니라 당신 환경에 맞춰집니다.
> - **코덱스 홈** = `$CODEX_HOME`(또는 자동 탐지해 기억한 경로), 기본 `~/.codex`
> - **브릿지 홈** = `$CODEX_BRIDGE_HOME`, 기본 `~/.codex-bridge`
> - **Claude 설정 폴더** = `$CLAUDE_CONFIG_DIR`, 기본 `~/.claude`

## 읽는 데이터 (모두 로컬)

- **코덱스 대화 기록** — 코덱스 홈 아래 `sessions/**/rollout-*.jsonl`, `session_index.jsonl`. 대시보드에 최근 대화를 보여주고 연결할 세션을 찾기 위해 평소 **읽기만** 합니다. (예외: 당신이 직접 "영구 삭제"를 누르면 그 파일을 지웁니다 — 아래 "데이터 삭제" 참고.)
- **코덱스 홈 경로 탐지** — `codex doctor` 출력에서 홈 경로 한 줄을 읽어 브릿지 홈의 `codex-home.txt`에 기억합니다.
- **코덱스 모델 목록 캐시** — 코덱스 홈 아래 `models_cache.json`. 모델·생각강도 선택 목록을 (하드코딩 대신) 당신 계정 기준으로 보여주기 위해 읽습니다.
- **Claude Code 대화 기록** — 완료 시점에 검증 가드가 훅이 알려준 대화기록 파일(`transcript_path`)을 읽어, 이번 턴에 실제로 코드 변경·검증이 있었는지 확인합니다. 또한 대시보드의 "두뇌 설정 어긋남" 경고를 위해 **지금 진행 중인 대화(없으면 이 폴더의 최근 대화)**의 `message.model`(실제 응답 모델)을 **읽기만** 합니다(신선도 창 밖의 오래된 기록은 제외).
- **Claude 설정(`settings.json`)의 모델** — 설정한 모델과 위 최근 응답 모델이 다른지("두뇌 설정 어긋남") 비교하려고 `model` 값을 **읽기만** 합니다(이 확장은 당신의 모델/생각강도를 절대 바꾸지 않습니다).
- **Claude 홈 경로 탐지** — 훅이 받은 `transcript_path`에서 Claude 설정 폴더를 도출해 브릿지 홈의 `claude-home.txt`에 기억합니다(확장이 `CLAUDE_CONFIG_DIR`을 직접 못 보는 환경 대응 — 코덱스의 `codex-home.txt`와 대칭).
- **검증 답변이 인용한 파일** — 코덱스 답변에 나온 `파일:줄` 근거가 실제로 존재하는지 확인하려고 그 로컬 파일을 열어 **줄 수만** 셉니다(내용을 어디로 보내지 않음).
- **검증 통계용 사용량** — 검증 통계 탭이 (a) 이 폴더 Claude 대화기록의 `message.usage`(입력·출력·캐시 토큰)와 `cwd`·`isSidechain`·`timestamp`를 읽어 **이 폴더 28일 토큰·턴수**를, (b) 연결 코덱스 세션 rollout의 `token_count`(`last_token_usage`/`total_token_usage`)와 `turn_context`(모델·추론강도)를 읽어 **검증 토큰·모델·강도**를 집계합니다. 모두 **로컬 집계**이고 수치만 씁니다(대화 원문 아님).
- **작업 폴더의 git 상태** — 완료 시점에 실제 변경이 있었는지 보려고 `git status`(작업트리 변경 목록)를 읽습니다.
- **현재 작업 맥락** — Claude Code 훅이 주는 작업 폴더·세션 ID(`CLAUDE_PROJECT_DIR`, `CLAUDE_CODE_SESSION_ID`).

## 쓰는 데이터 — ② 브릿지 홈 밖에서 일어나는 일 (정직하게)

- **Claude 설정 폴더의 `settings.json`** — 설치기가 검증·게이트 훅 4개를 병합합니다. 먼저 `.bak` 백업을 만들고, 형식이 예상과 다르면 손대지 않고 중단합니다. (훅 제거 시에도 이 파일을 고칩니다.) **확장(마켓 설치)도 같은 병합을 할 수 있지만, 알림 → 변경 내용(파일·백업 위치·훅 4줄) 확인 → [설치] 동의 1클릭 뒤에만** 하며 규칙은 설치기와 동일합니다(자동·무동의 변경 없음).
- **코덱스 홈의 `hooks.json`** — Codex↔Codex 구현 생명주기를 받기 위해 확장이 `SessionStart`·`UserPromptSubmit`·`PostToolUse`·`Stop` 명령 4개를 병합합니다. 실제 위치는 자동 탐지된 `$CODEX_HOME/hooks.json`(기본 `~/.codex/hooks.json`)입니다. 설치 동의창에서 정확한 경로를 먼저 보여 주며, 기존 파일은 `hooks.json.bak.<시각>`으로 백업하고 다른 사용자 훅은 보존합니다. 파일 형식이 예상과 다르거나 소유권 표식을 안전하게 기록하지 못하면 원본을 복원하고 중단합니다. Codex 설정 → Hook의 별도 신뢰 승인은 사용자가 직접 해야 합니다.
- **브릿지 홈에 스크립트 복사** — 설치 시 브릿지 스크립트(훅·브릿지 본체)를 브릿지 홈으로 복사합니다. **마켓 설치 시엔 확장이 같은 스크립트를 자동 배치**합니다(자체 폴더라 다른 앱 설정 안 건드림 — 레포 설치기를 쓴 상태면 덮지 않고 존중).
- **코덱스 홈 아래 `sessions/**/rollout-*.jsonl` 삭제** — 평소엔 안 지우지만, 당신이 "영구 삭제"를 눌러 확인하면 그 한 파일을 지웁니다(되돌릴 수 없음).
- **임시 파일(`os.tmpdir()`)** — **직접 검증(ask)**은 코덱스에 보낼 프롬프트를 디스크에 쓰지 않고 표준입력(stdin)으로 전달하며, 받을 출력만 잠깐 임시 폴더의 파일을 거친 뒤 읽고 곧 지웁니다. **내구 검증(ask-start/ask-wait)**은 예외 — 외부 도구 실행창이 닫혀도 검증이 이어지도록 프롬프트와 응답을 하네스 홈 `ask-jobs/`에 저장합니다(위 표 참조 — **7일 보존 후 하루 1회 정리에서 자동 삭제**, `ask-job clear <id> --confirm`으로 즉시 삭제 가능).

## 외부로 나가는 것

- **기본: 없음.** 별도 서버·분석·원격 수집이 없습니다. 코덱스 호출은 당신의 `codex` 명령을 로컬에서 실행하는 것뿐이며, 그때 코덱스가 OpenAI와 통신하는 것은 **코덱스 자체의 동작**이지 이 도구가 따로 보내는 것이 아닙니다. (탐색의 기본 정찰 — `scope-scout-self` 스크립트 — 도 같은 원리: 당신의 `claude` 명령을 로컬 실행하는 것이고, 그때의 통신은 Claude Code 자체의 동작입니다. 탐색 담당을 **Codex 정찰**(`scope-scout-codex`)로 선택한 경우도 동일합니다 — 당신의 `codex` 명령을 검증과 분리된 독립 실행 1회로 로컬에서 돌리는 것이고, 그때 자료 꾸러미가 코덱스를 통해 전달되는 통신은 코덱스 자체의 동작입니다.)
- **연결 점검(키 등록 시 한정)**: DeepSeek 키가 등록된 상태에서 대시보드에서 3트랙을 켜(저장하)면, 연결이 실제로 되는지 **점검 요청 1회(ping)**가 DeepSeek 주소로 전송됩니다 — 자료 꾸러미가 아니라 접속 확인용 최소 요청이며, 결과(정상/실패)는 알림으로 사실대로 표시됩니다. 키가 없으면 아무것도 전송되지 않습니다.
- **나머지 예외 — DeepSeek 지도 생성이 실행될 때**: `scope-scout-deepseek` 스크립트(또는 `deepseek-bridge map`)가 실행되면 그 순간 자료 꾸러미가 `deepseek.json`의 주소(기본 api.deepseek.com)로 전송됩니다. 실행 경로는 두 가지 — **당신이 직접**, 또는 **3트랙 자동 지시를 받은 Claude가**(아래 "자동 지시" — 키 등록=동의). 지도 생성 전송은 확장·훅이 직접 수행하지 않으며(위 연결 점검 1회만 확장이 트리거), 대시보드를 보는 것만으로는 전송이 일어나지 않습니다.
- **자동 지시(현재 동작)**: 3트랙 프로젝트에서 **지도가 없거나 낡았을 때(생성 이후 변경 신호 — 근거 파일 변경/삭제·새 커밋·작업트리 변경의 합산)**, 하네스가 턴 시작 시 Claude에게 "지도를 갱신하라"는 **지시문만** 넣습니다 — 같은 상태에는 1회, 단 같은 지도라도 **낡음 정도가 크게 진행되면(변경 신호 합산이 대략 2배 단위로 늘 때: 1→2→4→8…) 그때마다 다시 1회**(줄어들 때는 다시 알리지 않음 · 시간 기준 없음). 근거 파일 기록이 없는 구버전 지도는 낡음을 단정하는 대신 "판정 불가 — 재생성 권고"로 1회 알립니다. 실행·전송의 주체는 여전히 Claude가 돌리는 명령이며(이 확장·훅이 직접 전송하지 않음), 기본은 기본 정찰(Claude 겸임 — 별도 과금 없음·쓰시던 사용량 범위)입니다. **고급설정에 키를 등록하는 행위는 DeepSeek 정찰 자동 사용에 대한 동의**로 간주되어, Claude가 비교가 필요하다고 판단하면 그 키로 호출할 수 있습니다. 키를 삭제하면 DeepSeek 자동 사용도 함께 꺼집니다. 이 조건이 바뀌면 이 문서를 먼저 갱신합니다.
  - **보내는 것**: 작업트리 변경 diff(아래 제외 적용 후)·바뀐 식별자와 그 식별자를 참조하는 파일 **경로** 목록·과거 '함께 변경' 통계(파일 경로·횟수)·테스트 목록·최근 검증 실패 **메타**(시각·종류·요약 — 같은 대상 레포를 정찰하는 세션 폴더들 소유분 포함: 계약의 scoutRepo 역추적)·stable MAP(있으면). **무이력 모드(변경 기록 없는 폴더)에서는 diff가 없어 대신 최근 수정 파일(최대 8개)의 내용 앞부분(각 4,000자 상한) 발췌가 꾸러미에 실립니다 — 즉 이 모드에서 정찰을 실행하면 파일 본문 일부가 전송 대상에 포함됩니다**(민감 범주 파일은 이름 기준으로 목록에서 제외). 참고: 검증(Codex) 요청에 동봉되는 '정찰 관찰 신호'는 관찰 일지에서 파생한 **수치 요약뿐**(원문 추가 저장·전송 없음 — 일지 자체는 로컬). 3트랙에서는 검증(Codex) 요청에 **'결합 확인 요청' 블록**(관찰 일지의 기계 확인 가능 결합 후보 최대 3건 — 항목 원문·6자 식별자)이 함께 동봉될 수 있습니다: 검증자에게 '실제로 확인/반박한 경우에만' 답에 표기하도록 지시하는 채널이며(표기는 검증자의 자기보고 — 장부 승격·강등 재료로는 그 항목 경로의 실제 인용이 함께 요구됩니다), 전송 경로는 기존 검증 경로(당신이 이미 쓰는 Codex CLI) 그대로입니다(새 외부 통신 아님).
  - **안 보내는 것(자동 제외)**: `.env`류·키/토큰/credential/secret 범주 파일·인증서/개인키(.pem·.key 등)·`node_modules`/`dist`/`build`/`vendor`의 diff는 꾸러미를 만들 때 통째로 빠지고, 제외했다는 사실이 꾸러미에 표기됩니다. 전체 파일 무차별 전송은 하지 않습니다(git 모드=diff와 경로·통계 / 무이력 모드=위 고지대로 최근 수정 파일의 앞부분 발췌까지). **한계(정직)**: 제외는 파일 **이름의 범주** 기준입니다 — 평범한 이름의 파일(예: config.yaml) 안에 직접 적어둔 비밀값까지 찾아내 가려주지는 못하므로, 그런 값은 파일을 나눠 보관하는 것이 안전합니다.
  - **API 키 원문**은 요청 인증 헤더로만 쓰이고 화면·로그·꾸러미 어디에도 찍히지 않습니다.

## 모델에게 전달되는 내용

- **Claude에게**: 매 입력마다 당신이 정한 고정 규칙(계약)과, 검증 모드일 때 2트랙 지시 텍스트가 붙습니다.
- **코덱스에게**: 검증 요청 시 기본 지침과 당신이 보낸 검증 본문이 전달됩니다. (당신이 무엇을 검증 요청했는지가 그대로 코덱스로 갑니다.)

## 데이터 삭제

- **확장 제거 시 자동 정리(마켓 설치)**: 확장을 제거하면 다음 VS Code 재시작 시점에 **확장이 설치한 것만** 정리됩니다 — Claude 설정 훅 4개와 소유권 표식에 기록된 모든 Codex `hooks.json`의 Codex Peek 훅 4개(각각 백업 생성 후 제거, 다른 훅 보존), 확장이 배치한 브릿지 스크립트·표식. 훅 파일·소유권 표식이 손상돼 제거 완료를 확인할 수 없으면 실행 대상이 사라지는 고아 훅을 만들지 않도록 브릿지 런타임을 보존합니다. **연결·계약·통계 등 데이터는 보존**됩니다. 레포 설치기(install.js)로 설치한 훅·스크립트는 건드리지 않습니다.
- 자체 데이터 전부 지우기: 브릿지 홈(기본 `~/.codex-bridge`) 폴더 삭제
- 검증 훅 제거: 설치기의 훅 제거 경로 사용, 또는 Claude 설정 폴더의 `settings.json`과 Codex 홈의 `hooks.json`에서 해당 훅 삭제(설치 시 만든 `.bak` 참고)
- 확장 제거: VS Code에서 Codex Bridge 제거
- 코덱스/Claude 원본 기록은 이 도구가 만든 것이 아니므로 각 도구의 정책을 따릅니다.
